# FORMA — Архитектура проекта

---

## Бизнес-контекст

### Проблема
Большинство сайтов автосервисов выглядят шаблонно: белый фон, стоковые фото, прайс-листы, формы-конструкторы. Это не работает для премиального сегмента, где клиент выбирает не по цене, а по доверию и ощущению качества. Владелец автомобиля за 5-15 млн рублей не оставит заявку на сайте, который выглядит как сайт шиномонтажа.

### Решение
Лендинг, который позиционирует PDR-студию как ателье по восстановлению кузова, а не как «автосервис». Сайт должен вызывать то же чувство, что и сам процесс PDR: точность, терпение, невидимый результат.

### Целевая аудитория
- Владельцы автомобилей премиум- и бизнес-класса (BMW, Mercedes, Porsche, Lexus)
- Возраст 30-55, ценят время и качество больше, чем экономию
- Привыкли к премиальному сервису, чувствительны к визуальному качеству
- Часто выбирают по рекомендации или первому впечатлению от сайта

### Ключевые метрики успеха
- Конверсия в заявку (форма / звонок / мессенджер)
- Время на сайте > 2 минут (вовлечённость через scroll-анимацию)
- Bounce rate < 40% (качественный первый экран)

## Дизайн-решения и их обоснование

### Почему тёмная тема?
- **Контекст продукта**: PDR-мастер работает в тёмном боксе с LED-лампой. Тёмный сайт = атмосфера мастерской
- **Восприятие премиальности**: тёмные интерфейсы ассоциируются с luxury-брендами (Rolex, Vertu, Bang & Olufsen)
- **Техническая причина**: фотографии кузова с LED-полосами сняты на чёрном фоне — тёмная тема позволяет бесшовно интегрировать их

### Почему Georgia (serif) для заголовков?
- Serif-шрифты ассоциируются с традицией, мастерством, вниманием к деталям
- Georgia — системный шрифт, не требует загрузки (0 KB), хорошо рендерится на всех устройствах
- Контраст serif-заголовков и sans-serif-текста создаёт визуальную иерархию без дополнительных приёмов

### Почему приглушённое золото (#c9a96e), а не яркое?
- Яркое золото (#FFD700) выглядит дёшево на экране — ассоциация с «распродажей»
- Приглушённый тон ближе к цвету латуни/бронзы — материалы, которые ассоциируются с ручной работой
- На тёмном фоне даёт достаточный контраст, но не кричит

### Почему scroll-driven анимация на 64 кадрах?
- **Уникальность**: ни один конкурент не показывает процесс PDR так наглядно
- **Вовлечённость**: пользователь сам управляет анимацией — ощущение контроля увеличивает время на сайте
- **Образовательная функция**: большинство клиентов не знают, что такое PDR. Анимация объясняет без текста
- **Референс**: Apple использует аналогичную технику для iPhone/MacBook — аудитория знакома с паттерном

### Почему один HTML-файл без фреймворков?
- **Скорость разработки**: нет настройки сборки, зависимостей, конфигов
- **Простота деплоя**: один файл = можно захостить где угодно, включая простой файловый хостинг
- **Производительность**: нет overhead от React/Vue/Next.js — чистый DOM, минимальный JS
- **Портируемость**: легко передать клиенту или другому разработчику

## Процесс создания

### Этап 1: Структура и контент
Создана базовая структура лендинга с 13 секциями. Определена дизайн-система (палитра, типографика, компоненты). Написаны тексты, ориентированные на premium-аудиторию.

### Этап 2: Canvas-анимация
1. Исходные фотографии: «до» (вмятина с LED-полосами) и «после» (ровная панель)
2. Генерация видео через Google Flow (image-to-video) — AI-модель интерполировала промежуточные состояния ремонта
3. Извлечение 64 кадров из видео (Python + OpenCV): 32 кадра реверса + 32 кадра оригинала → цикл smooth → dent → smooth
4. Конвертация в base64 JPEG (960px, quality 75-82) для инлайна в HTML
5. Реализация scroll-driven логики с lazy-loading и прелоадером

### Этап 3: Hero-секция
Фоновое изображение hero берётся из первого кадра canvas-анимации (ровная панель с LED-полосами). Это обеспечивает визуальную связь между hero и анимацией при скролле. Добавлены многослойные оверлеи: затемнение для читаемости текста + золотой радиальный блик для премиального ощущения.

### Этап 4: Интерактивные компоненты
BA-слайдер с поддержкой mouse и touch. Fade-in анимации через Intersection Observer. Smooth scroll навигация.

### Этап 5: SEO и мета
Schema.org разметка (AutoRepair), Open Graph, favicon, подготовка под Яндекс.Метрику.

### Проблемы и решения
| Проблема | Решение |
|----------|---------|
| AI сгенерировала видео в обратном порядке (вмятина → ровная вместо ровная → вмятина) | Реверс кадров + склейка: reverse + original = полный цикл |
| 64 кадра по 260 KB = 16 MB | Ресайз до 960px + JPEG quality 75 → 60 KB/кадр, итого ~4 MB |
| Долгая загрузка из-за 64 кадров | Lazy-load: 8 критических кадров сразу, остальные батчами + прелоадер |
| Hero выглядел пустым (текст на тёмном фоне) | Фоновое фото из первого кадра canvas + многослойные оверлеи |
| Вотермарка Veo от Google Flow | Пока не решено — требует перегенерации видео или платной версии |

## Обзор

Single-page лендинг в одном HTML-файле (5.81 MB). Без фреймворков, без сборщиков, без внешних зависимостей. Всё инлайн: стили, скрипты, изображения (base64).

**Стек:** HTML5 + CSS3 + Vanilla JS
**Позиционирование:** премиальный сервисный бренд (quiet luxury)

## Секции — детальное описание

### 1. Preloader (#preloader)

- Полноэкранный оверлей (position:fixed, z-index:9999)
- Логотип + прогресс-бар + процент загрузки
- Показывает прогресс загрузки кадров canvas-анимации
- Исчезает с opacity transition после загрузки

### 2. Nav

- `position:fixed` — всегда наверху
- `backdrop-filter:blur(12px)` — размытие фона
- Золотая нижняя граница (rgba)
- Логотип слева, меню справа
- На мобильных: меню скрыто (`display:none` при `<900px`)

### 3. Hero

- `height:100vh` — на весь экран
- **Фоновое фото** (.hero-bg): `position:absolute`, изображение из первого кадра canvas, `filter:brightness(0.35)`
- **Оверлей** (::before): двойной градиент — линейный (затемнение слева) + радиальный (золотой блик)
- **Золотой блик** (::after): `radial-gradient`, `filter:blur(60px)`, 500x500px
- Контент (`z-index:2`): бейдж → заголовок → подзаголовок → 2 кнопки

### 4. Stats

- CSS Grid: `repeat(4, 1fr)`
- Разделители: `border-right` + `border-top/bottom`
- Числа: Arial 52px, золотые
- На мобильных: `repeat(2, 1fr)`

### 5. Frame Sequence (Scroll-driven анимация)

**Самый сложный компонент.** Управляется скроллом, показывает процесс ремонта вмятины.

**Структура:**
```
.frame-seq-section
└── .frame-seq-scroll-space (height: 500vh — задаёт длину скролла)
    └── .frame-seq-sticky (position:sticky, top:0, height:100vh)
        ├── #frame-container (64 <img> тегов, position:absolute)
        ├── .frame-seq-overlay (градиент сверху/снизу)
        ├── .frame-seq-scroll-hint (подсказка «прокрутите»)
        ├── .frame-seq-label (заголовок + текст)
        ├── .frame-seq-progress (индикатор)
        └── .frame-seq-bar (прогресс-бар)
```

**Данные:** 64 кадра в массиве `const frames = [...]` (base64 JPEG, ~100 KB каждый)

**Загрузка (lazy):**
1. Первые 8 кадров загружаются сразу (critical)
2. Остальные — батчами по 8 в фоне
3. Preloader обновляется по мере загрузки

**Логика скролла:**
```js
progress = (scrollY - sectionTop) / (sectionHeight - winH)  // 0..1
frameIdx = Math.floor(clamped * (totalFrames - 1))           // 0..63
```
- Линейная интерполяция: скролл 0% = кадр 0, скролл 100% = кадр 63
- Throttled через `requestAnimationFrame`

**Последовательность кадров:**
- 0-31: ровная поверхность → максимальная вмятина (реверс видео)
- 32-63: максимальная вмятина → ровная поверхность (оригинальное видео)

### 6. Before/After (BA-слайдер)

- Два `<img>` (до/после) наложены друг на друга
- `.ba-after` обрезается через `clip-path: inset(0 ${100-pct}% 0 0)`
- Разделитель (`.ba-divider`) + золотой круг-ручка (`.ba-handle`)
- Перетаскивание: `mousedown/mousemove/mouseup` + `touchstart/touchmove/touchend`
- Метки «ДО» / «ПОСЛЕ»

### 7. Services

- CSS Grid: `repeat(3, 1fr)`, gap:2px
- Карточки (.service-card): hover-эффект с золотой полоской сверху (`::before`, `transform:scaleX`)
- Последняя карточка — CTA «Оценка за 15 минут»

### 8. Why (Преимущества)

- 2 колонки: список преимуществ + визуальный блок
- Каждый пункт: иконка (◈) + заголовок + описание
- Визуальный блок: тёмный фон с SVG-линиями (opacity:0.15)

### 9. Process (Шаги работы)

- CSS Grid: `repeat(5, 1fr)`
- Горизонтальная линия (::before) соединяет шаги
- Каждый шаг: круг с номером → заголовок → описание
- На мобильных: `repeat(1fr, 2)`

### 10. Gallery

- CSS Grid: `repeat(3, 1fr)`, gap:3px
- `aspect-ratio:4/3`
- Hover: масштабирование (scale 1.05) + оверлей + подпись

### 11. Reviews (Отзывы)

- CSS Grid: `repeat(3, 1fr)`
- Карточка: звёзды → цитата (italic) → автор (аватар + имя + авто)
- Аватар: первая буква имени в золотом круге

### 12. Contacts

- 2 колонки: информация + форма
- Информация: телефон, мессенджеры, адрес, часы работы
- Форма: имя + телефон + textarea + кнопка
- Яндекс.Карта: iframe, `filter:grayscale(0.8) brightness(0.7)`

### 13. Footer

- Flexbox: логотип | копирайт | описание
- Фон: #050505

## JavaScript — модули

Весь JS в одном `<script>` блоке, разделён комментариями:

### 1. FRAME SEQUENCE
- `frames[]` — массив base64 строк (64 элемента)
- `loadFrame(i)` — создаёт `<img>`, добавляет в DOM, возвращает Promise
- `showFrame(idx)` — активирует кадр (добавляет class `active`, убирает у остальных)
- `updatePreloader()` — обновляет прогресс-бар и процент
- Lazy-loading: критические 8 кадров → батчи по 8

### 2. SCROLL HANDLER
- `updateFrameSeq()` — вычисляет прогресс скролла, выбирает кадр
- `requestAnimationFrame` throttling
- Обновляет прогресс-бар и подсказку скролла

### 3. BA SLIDER
- Drag-события: mouse + touch
- `setPos(x)` — вычисляет процент, обновляет `clip-path`, `left` разделителя

### 4. SMOOTH SCROLL
- Обработчик кликов по навигации и CTA-кнопкам
- `scrollIntoView({behavior:'smooth'})`

### 5. FADE-IN
- `checkFadeIn()` — проверяет все `.fade-in` элементы
- Добавляет `.visible` когда элемент виден на 15% (getBoundingClientRect)
- Слушает scroll и resize

## Как адаптировать под другой проект

### 1. Дизайн-система (5 минут)
Поменять CSS-переменные в `:root`:
```css
/* Кофейня */
--gold: #8B6C42;    /* тёплый коричневый */
--gold2: #A68B5B;
--dark: #1a1510;    /* тёплый чёрный */
--dark2: #201a14;
--dark3: #2a2118;
--text: #f0e6d8;    /* кремовый */
--muted: #9a8b7a;
```

### 2. Типографика
Заменить `Georgia` на нужный шрифт в `body` и заголовках. Подключить через Google Fonts если нужно.

### 3. Контент
- Заменить тексты в HTML (заголовки, описания, метки)
- Заменить изображения в BA-слайдере и галерее
- Обновить Schema.org `@type` и данные

### 4. Секции
- Удалить ненужные (например, Frame Sequence для кофейни — overkill)
- Добавить специфичные (меню, бронирование столика)

### 5. Canvas-анимация
Если не нужна — удалить:
- CSS: `.frame-seq-*` блок (строки 174-235)
- HTML: `<!-- FRAME SEQUENCE -->` секция
- JS: `// FRAME SEQUENCE` и `// SCROLL HANDLER` блоки
- Массив `frames[]` (это ~8 MB данных)

Это уменьшит файл с ~9 MB до ~1 MB.

## Ограничения и технический долг

1. **Размер файла** — 5.81 MB из-за инлайн base64 кадров. Решение: вынести кадры в отдельные .jpg файлы
2. **Одна точка отказа** — всё в одном файле. Решение: разделить на HTML + CSS + JS + images
3. **Нет сборки** — нет минификации, tree-shaking. Для продакшна нужен хотя бы минификатор
4. **Нет серверной части** — форма обратной связи не отправляет данные
5. **Вотермарка Veo** — в кадрах canvas и hero-фоне (от Google Flow)
6. **Плейсхолдеры** — телефон, адрес, мессенджеры, Метрика, фото галереи, отзывы
7. **Accessibility** — нет aria-labels, alt-текстов, skip-navigation. Требует доработки для WCAG 2.1
8. **Мобильное меню** — скрыто на <900px, но нет бургер-меню (навигация только через скролл)

## Что отличает этот проект

1. **Scroll-driven анимация из AI-видео** — уникальный пайплайн: реальное фото → Google Flow (image-to-video) → извлечение кадров → scroll-интерактив. Ни один конкурент в PDR-нише не использует подобное
2. **Zero-dependency** — весь проект в одном файле без единой внешней зависимости, при этом включает: lazy-loading, scroll-driven анимацию, drag-слайдер, responsive, SEO, structured data
3. **Дизайн-система для переиспользования** — CSS-переменные + компонентные классы позволяют адаптировать проект под любую сферу за минуты
