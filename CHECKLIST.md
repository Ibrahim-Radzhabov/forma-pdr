## Анализ расхождений между оценками

| Критерий | Claude | GPT 5.4 | Расхождение |
|----------|--------|---------|-------------|
| Визуальный дизайн | 8 | 8 | Совпали. Оба отмечают попадание в quiet luxury, но незавершённость деталей |
| UX и архитектура | 8.5 | 7 | GPT строже: указывает на потерю темпа конверсии и слабый progressive conversion path |
| Техническая архитектура | 7 | 6 | GPT точнее: base64-lazy-load — не настоящий network lazy-load, пользователь всё равно качает 5.81 MB |
| Креативные решения | 9 | 8 | Claude щедрее; GPT предупреждает о риске «слишком Apple-inspired» без собственного лица |
| SEO | 7.5 | 6 | GPT глубже: указывает на отсутствие Twitter Cards, geo, sameAs, FAQ schema |
| Готовность к запуску | — | 5 | Claude не выставил отдельную оценку, GPT — жёсткое «5/10» |
| **Итого** | **8** | **7** | GPT строже на 1 балл, но аргументация конкретнее |

### Где GPT 5.4 точнее Claude:
- **Lazy-load — фейковый.** Claude похвалил стратегию «8 критических + батчи по 8», но GPT верно заметил: раз base64 инлайн в HTML, браузер скачивает ВСЁ сразу. Lazy-load откладывает только декодирование, а не передачу по сети. Это ключевое замечание
- **Progressive conversion path.** GPT указал, что между hero и финальной формой нет промежуточных точек конверсии — пользователь может «красиво проскроллить и уйти»
- **Legal layer.** GPT единственный упомянул consent на обработку данных, политику конфиденциальности — обязательные для РФ (152-ФЗ)
- **Дрейф документации.** GPT заметил расхождение: в ARCHITECTURE.md заявлен Intersection Observer, а в коде — ручной checkFadeIn() на scroll/resize

### Где Claude точнее GPT 5.4:
- **Конкретные рекомендации по шрифтам** — назвал Cormorant Garamond, Playfair Display, Inter, Manrope (GPT говорил абстрактно)
- **Sticky CTA на мобильных** — конкретная рекомендация, которую GPT не озвучил
- **Кликабельный телефон в nav** — практичный совет для мобильной конверсии

### Что оба упустили:
- **Производительность скролла на слабых устройствах** — 64 img-тега в DOM, все с position:absolute. На бюджетных Android это может тормозить
- **Prefers-reduced-motion** — GPT упомянул вскользь, Claude не упомянул. Для accessibility это важно
- **Контраст текста на hero** — белый текст на затемнённом фото может не пройти WCAG AA в некоторых зонах
- **Print styles** — для premium-клиента, который может захотеть распечатать контакты

## Объединённый чеклист

- [ ] **Заполнить ВСЕ плейсхолдеры** — телефон, адрес, WhatsApp, Telegram, ID Яндекс.Метрики. Один фейковый телефон обнулит доверие всего лендинга
- [ ] **Реальные фото в галерею** — заменить «Фото 1–6» на настоящие кейсы с подписями (марка авто, тип повреждения, время работы)
- [ ] **Бургер-меню на мобильных** — навигация полностью скрыта на <900px. 60-70% трафика локального бизнеса — мобильный
- [ ] **Подключить форму** — реальная отправка (Telegram-бот, email, CRM). Сейчас форма не работает
- [ ] **Добавить og:image** — без него шеринг в WhatsApp/Telegram (основной канал ЦА) выглядит безлико. Размер: 1200x630px
- [ ] **Политика конфиденциальности + consent** — обязательно по 152-ФЗ. Чекбокс «Согласен на обработку данных» в форме
- [ ] **Реальные отзывы** — заменить шаблонные на настоящие (или убрать секцию до получения реальных)

### 🟡 Серьёзные доработки (нужно до запуска)

#### Производительность
- [ ] **Вынести base64 кадры в отдельные .jpg файлы** — текущий «lazy-load» фейковый: браузер всё равно скачивает 5.81 MB HTML целиком. Отдельные файлы = реальный lazy-load + кэширование + CDN
- [ ] **Разделить HTML / CSS / JS** — минимум 3 файла для нормального кэширования
- [ ] **Минифицировать CSS и JS** — хотя бы базовая минификация перед деплоем
- [ ] **Рассмотреть canvas + drawImage вместо 64 img-тегов** — меньше DOM-элементов, лучше производительность на слабых устройствах

#### UX и конверсия
- [ ] **Sticky CTA на мобильных** — плавающая кнопка «Записаться» или телефон, появляется после скролла за hero
- [ ] **Кликабельный телефон в nav** — `<a href="tel:+7...">` для мобильной конверсии
- [ ] **Промежуточные CTA** — между hero и контактами 13 секций. Добавить inline CTA после process или reviews: «Оценить по фото в WhatsApp»
- [ ] **Micro-proof под stats** — «оценка 15 минут / 1–4 часа / без перекраса» — быстрые ответы на главные вопросы
- [ ] **Reassurance у формы** — «Ответим за 15 минут», «Можно прислать фото в мессенджер»
- [ ] **BA-слайдер — добавить подпись** — «Дверь BMW X5 / без покраски / 3.5 часа» — делает доказательство конкретным

#### Accessibility
- [ ] **Alt-тексты на все изображения** — это и SEO, и WCAG
- [ ] **Aria-labels** — на интерактивных элементах (BA-слайдер, форма, навигация)
- [ ] **Skip-navigation link** — для keyboard-пользователей
- [ ] **Keyboard navigation на BA-слайдере** — сейчас работает только mouse/touch
- [ ] **Prefers-reduced-motion** — отключать fade-in и scroll-анимацию для пользователей с настройкой
- [ ] **Проверить контраст текста на hero** — белый на затемнённом фото может не пройти WCAG AA

#### SEO
- [ ] **Canonical URL** — `<link rel="canonical">`
- [ ] **Twitter Card мета-теги** — twitter:card, twitter:title, twitter:description, twitter:image
- [ ] **Alt-тексты** (дублирует accessibility — но критично для SEO)
- [ ] **Расширить Schema.org** — добавить: image, geo, sameAs, areaServed, serviceType, hasMap
- [ ] **FAQ schema** — если добавить FAQ-секцию, разметить FAQPage для rich snippets
- [ ] **Robots.txt + sitemap.xml** — при деплое на продакшн
