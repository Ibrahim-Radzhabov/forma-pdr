# FORMA PDR

## Что это
Лендинг премиальной PDR-студии (беспокрасочное удаление вмятин) в Москве -- один HTML-файл без фреймворков, с canvas scroll-анимацией процесса ремонта.

## Стек
HTML5, CSS3, Vanilla JS, Canvas API. Нулевые зависимости -- ни npm, ни бандлеров, ни внешних шрифтов.

## Архитектура
- **Single HTML file** (`forma_landing_v3.html`, ~1260 строк, ~5.8 MB) -- стили, скрипты и base64-кадры инлайн.
- **64-frame canvas scroll animation** -- пользователь скроллит, кадры переключаются. Кадры 0-31: ровная поверхность -> вмятина; 32-63: вмятина -> ровная. Lazy-load: 8 критических сразу, остальные батчами по 8. Прелоадер показывает прогресс.
- **Before/After slider** -- два img наложены, clip-path управляется drag (mouse + touch).
- **Дизайн-система** -- CSS-переменные: --gold (#c9a96e), --dark (#0a0a0a), --text (#e8e4df). Georgia для заголовков, Arial для текста. Тёмная тема, quiet luxury.
- **SEO** -- Schema.org (AutoRepair, JSON-LD), Open Graph, SVG favicon (буква F).
- **Responsive** -- один breakpoint 900px. На мобильных сетки сворачиваются в 1-2 колонки.
- **JS-модули** (внутри одного script): frame sequence, scroll handler, BA slider, smooth scroll, fade-in (ручной checkFadeIn на scroll/resize, не Intersection Observer).

## Запуск
Открыть `forma_landing_v3.html` в браузере. Для разработки -- Live Server в VS Code.

## Важно
- Папка `frames/` содержит 64 JPEG-файла (frame_000.jpg -- frame_063.jpg) -- исходные кадры анимации. В HTML они встроены как base64, поэтому папка нужна только как исходник.
- `frames_base64.js` -- артефакт, не используется в продакшне.
- Все 64 кадра закодированы в base64 прямо в HTML (~5.8 MB). Из-за этого "lazy-load" откладывает только декодирование, а не сетевую загрузку.
- Fade-in реализован через ручной checkFadeIn() на scroll/resize, а не через Intersection Observer (в ARCHITECTURE.md указано неверно).
- Телефон, адрес, ссылки на мессенджеры -- плейсхолдеры. Форма обратной связи не отправляет данные.
- В hero-фоне и canvas-кадрах видна ватермарка Veo от Google Flow.
